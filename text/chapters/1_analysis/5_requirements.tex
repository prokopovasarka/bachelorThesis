\section{Požadavky}
% dělíme na funkční a nefunkční, požadavky obsahují následující údaje:
% název - stručný popis požadavku
% identifikátor/zkratka - snazší odkazování
% popis - nejdůležitější, obsahuje popis požadavku
% priority - důležitost požadavku při vývoji, nejnižší lze nesplnit - použijeme MoSCoW
% 1. must have - musí být splněný vždy, bez něj nefunguje app
% 2. Should have - důležité pro projekt, ale ne nezbytné
% 3. could have - nice to have
% 4. will not have - volitelné, ne priorita pro tento release, spíše pro budoucí verze

% Priorita: Must have
